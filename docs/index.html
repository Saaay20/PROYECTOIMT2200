El Problema de la Valoración

La compra-venta de viviendas exige que tanto compradores como vendedores cuenten con un valor de referencia objetivo. Actualmente, este proceso suele ser manual, costoso o subjetivo.

Nuestra Propuesta: Desarrollar una herramienta automática que estime el precio de mercado utilizando datos públicos, aprovechando la disponibilidad de información en portales inmobiliarios.

Pregunta clave: ¿Es posible predecir tasaciones de manera automática con alta precisión solo usando datos web?

Diferencia inicial de precios promedio (UF) entre departamentos y casas.

Metodología y Proceso ETL

Documentación del flujo de datos desde la extracción hasta el modelo.

1. Web Scraping

Extracción automatizada de datos desde portalinmobiliario.com. Se generaron datasets conteniendo estas viviendas y sus respectivas característica.

2. Geolocalización

Implementación de un proceso para convertir direcciones textuales en coordenadas precisas (Latitud/Longitud) para reducir el ruido.

3. Limpieza (ETL)

Eliminación de outliers, normalización de unidades monetarias (Pesos a UF) y limpieza de registros con datos faltantes o inconsistentes.

4. Modelado

Uso de Regresión Lineal para la predicción de precios.

Análisis Exploratorio de Datos (EDA)

Antes de entrenar el modelo, visualizamos las relaciones clave entre variables para entender el comportamiento del mercado.

Correlaciones del Mercado

Identificamos una fuerte correlación positiva entre la superficie construida y el precio, lo cual validó la selección de variables para la regresión.

Dispersión de Precios

La relación precio/superficie muestra heteroscedasticidad: la varianza aumenta en propiedades de mayor tamaño, lo que dificulta la predicción en el segmento de lujo.

Distribución Normalizada

Análisis de la densidad de precios por m² construido, permitiendo identificar la "zona media" del mercado y aislar outliers.

Impacto de las Amenidades

Evaluación de cómo características adicionales (piscina, jardín) influyen en el valor final.

Resultados del Modelo

El modelo mostró un desempeño sólido en casas, logrando un R² de 0.9. Los precios aproximados que se entregaban eran muy cercanos a los rangos reales de mercado.

Sin embargo, el rendimiento fue inferior en departamentos debido a la falta de variables clave para este tipo de vivienda (orientación, piso) que no estaban disponibles en la data pública.

Discusión, Fuentes de Error y Ética

Fuentes de Error

  • 1. Distorsión por "Ventas en Verde": En ciertos sectores, los precios de proyectos nuevos distorsionan los valores de reventa, generando estimaciones menos precisas.
  • 2. Escasez de Datos: En comunas con menos de 100 registros, el modelo produce predicciones poco representativas.
  • 3. Variables Ocultas: La baja performance en departamentos se debe a la ausencia de datos como "orientación" o "número de piso" en el scraping.

Ética y Proyecciones

Implicancias Éticas: Automatizar tasaciones puede perpetuar sesgos si los datos históricos reflejan discriminación de zonas. Además, una herramienta mal calibrada podría inflar artificialmente precios en barrios emergentes.

Trabajo Futuro: Es fundamental diseñar un scraping específico para el segmento de departamentos y actualizar los datos periódicamente para que el sistema refleje la evolución real del mercado inmobiliario.
