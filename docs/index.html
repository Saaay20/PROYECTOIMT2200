El Problema de la Valoración

La compra-venta de viviendas exige que tanto compradores como vendedores cuenten con un valor de referencia objetivo. Actualmente, este proceso suele ser manual, costoso o subjetivo.

Nuestra Propuesta: Desarrollar una herramienta automática que estime el precio de mercado utilizando datos públicos, aprovechando la disponibilidad de información en portales inmobiliarios.

Pregunta clave: ¿Es posible predecir tasaciones de manera automática con alta precisión solo usando datos web?

Diferencia inicial de precios promedio (UF) entre departamentos y casas.

Metodología y Proceso ETL

Construcción del Dataset: Del dato crudo a la información útil.

1. Web Scraping

Extracción automatizada desde portalinmobiliario.com. Se generaron datasets diferenciados para casas y departamentos.

2. Geolocalización

Script personalizado que convirtió direcciones textuales inexactas en coordenadas precisas (Latitud/Longitud) iterando URL por URL.

3. Limpieza (ETL)

Eliminación de outliers extremos, estandarización de divisas (CLP a UF) y limpieza de valores nulos o inconsistentes.

4. Preparación

Concatenación de bases de datos y selección de "features" relevantes (Superficie, ubicación, amenidades) para el entrenamiento.

Análisis Exploratorio de Datos (EDA)

Antes de modelar, realizamos una exploración visual para entender la distribución del mercado, validar hipótesis y detectar patrones ocultos.

1. Correlaciones del Mercado

Utilizamos un mapa de calor para identificar variables predictoras. Se confirmó una correlación positiva fuerte entre la superficie construida y el precio, lo cual justificó su uso como variable principal en la regresión.

2. Heteroscedasticidad

El gráfico de dispersión revela que a mayor tamaño de la propiedad, mayor es la varianza del precio. Esto indica que el segmento de lujo es más difícil de predecir que el segmento medio debido a la dispersión de los datos.

3. Distribución de Precios

Analizamos el histograma de precios por m² para entender la forma del mercado. Observamos una distribución con sesgo positivo, lo que nos ayudó a aislar outliers extremos que podían "ensuciar" el entrenamiento.

4. Valor de las Amenidades

Evaluamos variables categóricas. La presencia de piscina, jardín o quincho mostró un impacto medible en el precio promedio, validando la necesidad de incluir estas columnas en el modelo final.

Resultados del Modelo Predictivo

Casas: Éxito Rotundo

Logramos un R² de 0.9 en el conjunto de prueba.

El modelo de Regresión Lineal para casas demostró ser altamente efectivo. Las predicciones entregadas se mantuvieron consistentemente dentro de los rangos de mercado reales, validando la calidad de las variables seleccionadas (superficie + ubicación).

Departamentos: Desafíos

Rendimiento inferior al esperado.

A diferencia de las casas, el precio de los departamentos depende críticamente de variables que no siempre son públicas, como la orientación (Norte/Sur) y el número de piso. La ausencia de estos datos en el scraping limitó la precisión en este segmento.

Discusión, Fuentes de Error y Ética

Fuentes de Error

  • 1. Distorsión por "Ventas en Verde": En ciertos sectores, los precios de proyectos nuevos distorsionan los valores de reventa, generando estimaciones menos precisas.
  • 2. Escasez de Datos (Cold Start): En comunas con menos de 100 registros, el modelo produce predicciones poco representativas.
  • 3. Variables Ocultas: La baja performance en departamentos se debe a la ausencia de datos como "orientación" o "número de piso" en el scraping.

Ética y Proyecciones

Implicancias Éticas: Automatizar tasaciones puede perpetuar sesgos si los datos históricos reflejan discriminación de zonas (redlining digital). Además, una herramienta mal calibrada podría inflar artificialmente precios en barrios emergentes, contribuyendo a la gentrificación.

Trabajo Futuro: Es fundamental diseñar un scraping específico para el segmento de departamentos y actualizar los datos periódicamente para que el sistema refleje la evolución real del mercado inmobiliario.
