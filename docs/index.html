Tasación Inmobiliaria Automatizada

Un enfoque de Ciencia de Datos para predecir el valor de mercado de propiedades en la Región Metropolitana utilizando Web Scraping y Machine Learning.

Contexto y Motivación

La compra-venta de viviendas exige un valor de referencia objetivo. [cite_start]Actualmente, obtener una tasación precisa es lento o costoso[cite: 16]. [cite_start]Nuestro proyecto busca automatizar esta tasación aprovechando la información pública disponible en portales inmobiliarios[cite: 18].

Objetivo

[cite_start]Construir una herramienta capaz de estimar el precio de mercado (en UF) de una propiedad basándose en sus atributos (superficie, ubicación, características) y encontrar bienes comparables[cite: 22, 23].

Adquisición de Datos (Web Scraping)

Para construir nuestro dataset, implementamos un proceso de extracción automatizada desde PortalInmobiliario.com. El flujo de trabajo consistió en:

    [cite_start]
  1. Scraping Masivo: Extracción de datos separados para Casas y Departamentos[cite: 92].
    2. [cite_start]
  2. Geocodificación: Debido a la falta de coordenadas exactas, implementamos un script adicional que recorrió cada URL para extraer la dirección precisa y convertirla en Latitud/Longitud[cite: 44, 92].
    3. [cite_start]
  3. Limpieza: Unificación de datasets, eliminación de valores nulos, corrección de unidades monetarias y filtrado de outliers[cite: 822].
[cite_start]

Desafío Técnico: Se descartaron variables ruidosas como el "título de la publicación" para centrarse puramente en las coordenadas geográficas y atributos físicos[cite: 823].

Análisis Exploratorio de Datos (EDA)

Antes de modelar, analizamos la distribución de precios y las correlaciones. [cite_start]Descubrimos una alta dispersión en los precios, explicada fuertemente por la ubicación (comuna) más que solo por el tamaño[cite: 84].

Distribución de Precios por Comuna: Se observa la variabilidad del valor UF según el sector geográfico.
Matriz de Correlación: Identificación de las variables que más influyen en el precio final.

Modelamiento y Resultados

Utilizamos una Regresión Lineal Múltiple para predecir el precio. El modelo fue entrenado y validado con un set de datos separado.

0.90 R² (Coeficiente de Determinación)
92-99% Precisión en Casas

Visualización de la Regresión

Predicción vs. Valor Real: La línea roja representa la predicción perfecta. Los puntos azules muestran el desempeño de nuestro modelo. Se observa un ajuste muy preciso para la mayoría de los casos.

Conclusiones Clave

    [cite_start]
  • Éxito en Casas: El modelo es altamente confiable para estimar el valor de casas, superando incluso a algunas herramientas comerciales[cite: 831].
  • Desafío en Departamentos: El rendimiento fue menor en departamentos. [cite_start]Esto se debe a la falta de variables verticales (piso, orientación, gastos comunes) que no fueron capturadas en el scraping inicial[cite: 816, 817].
    • [cite_start]
  • Comparables: Se validó que el uso de KNN es efectivo para encontrar propiedades similares ("vecinos cercanos")[cite: 833].
